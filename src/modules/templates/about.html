What is This?

NPM by numbers is a visualization of data gathered from NPM.js. Specifically, it as an overview of the packages that existed in NPM as of Sept 22nd, 2014. This tool gathers information about version numbers, dependents, package age, last update dates, maintainer count and so on. Here you can explore how the entire NPM ecosystem looks like across those variables.

Where does the data come from?

The original dataset is actually the entirety of skimdb - NPMs database of package information without the binaries. You can see it here: https://skimdb.npmjs.com/registry.

The dataset was then reduced to extract aggregate information such as: The number of packages across different major versions, or how many dependents packages have when you break it down by age etc.

Who made this?

NPM By Numbers was made by Irene Ros at Bocoup. NPM By Numbers relies heavily on data from NPM but is not affiliated with NPM, Inc. We are very thankful to NPM for their support and assistance. Having this data available and open to all is a great example of why we love open source!

Why did you make this?

Many of us use NPM on a daily basis for granular tasks such as installing or publishing packages. Rarely do we have the opportunity to step out and explore the entire ecosystem of NPM. What kind of packages are on it? What is the publishing behaviour of contributors? What packages matter? How many packages are abandoned? These are just a few questions we could ask about NPM as a whole. Understanding the ecosystem we operate in is essential to making informed technology decisions. We are very happy that NPM is allowing us to explore this data and that we can tell the story of NPM together.

Is the code for this open source?

Of course! All the code behind NPM By Numbers is available on github. Specifically:

  • Data analysis - Instructions for replicating skimdb, extracting relevant data about packages and iPython notebooks that produce the final datasets.
  • Visualiation - All the code behind this tool. This visualization was built with Backbone.js, d3.js and d3.chart.
