Packages on npm declare a version for their releases. The version of the package is specified by Semantic Versioning. A version number is written like so: major.minor.patch.

A major version increment is a big deal in theory - packages should only increment a major version when when they introduce incompatible API changes. Most packages on npm (82%) have a major version of zero (0.*...).
